# VOICE-006 専用ホールドキーの音声でCodexに質問する

## 対象アプリ

Voice Input Local

## 対象リポジトリ

`/Users/pon/dev/mac-voice-typing`

## 目的

通常の音声入力とは別のキーを長押しして質問を話し、Codexの回答をアプリ内で確認できるようにする。

## 背景

現在はFnなどのホールド中に話した内容を入力欄へ挿入する機能だけがある。調べたい内容を音声でCodexへ送り、入力作業を中断せず回答を得られる経路が必要。

## やること

- Codex質問専用のホールドキーを追加する
- 既定キーを右Commandにする
- 専用キーを設定画面から変更可能にする
- 短いタップやCommandショートカットでは起動せず、長押し時だけ録音を始める
- 音声認識結果をCodex CLIへ送り、回答を専用ウィンドウに表示する
- 通常音声入力とCodex質問のキー競合を防ぐ
- 既存設定ファイルを壊さず新しい既定値を補完する
- Codexへ送信されるデータについてREADMEとプライバシー文書を更新する

## やらないこと

- 画面キャプチャの添付
- 音声データそのものの外部送信
- Codexによるファイル変更やコマンド実行
- 通常音声入力のFn既定値変更

## 完了条件

- 右Commandを長押しして話し、離すとCodexへ質問できる
- 右Commandの短いタップや通常のCommandショートカットでは録音しない
- 専用キーを右Option、Fn、右Command、右Controlから変更できる
- 通常音声入力キーと専用キーが同一にならない
- 回答、処理中、エラーがアプリ内に表示される
- 既存の通常音声入力に回帰がない

## 検証方法

- 長押し判定、既定設定、旧設定移行、Codex連携の失敗テストを先に追加する
- `swift test`
- `swift build`
- `./build-app.sh release`
- アプリを`open`で起動し、短押し、ショートカット、長押し質問、キー変更を確認する
